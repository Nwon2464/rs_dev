# 검색엔진 최적화와 SSG 전환 계획

## 목적

이 사이트는 한국과 일본의 사용자가 다음 정보를 검색으로 쉽게 찾도록 한다.

- 일반 장비 개방 옵션
- 비규격 장비 일반 옵션
- 비규격 장비 개방 옵션
- 비규격 장비 티어별 수치

현재 웹은 React SPA다. 브라우저가 CSV와 JSON을 받은 뒤에 화면을 그린다.
검색엔진은 처음 받은 HTML에서 실제 옵션 내용을 읽기 어렵다.
SSG(사이트를 배포하기 전에 HTML 페이지를 미리 만드는 방식)로 핵심 내용을
HTML에 넣어 이 문제를 해결한다.

## 확정된 기준

- 배포는 GitHub Pages를 계속 사용한다.
- 일본어가 기본 언어다.
- 한국어와 일본어는 서로 다른 주소를 사용한다.
- 데이터 기준은 항상 일본 클라이언트의 업데이트 시점이다.
- 데이터는 보통 2~3개월마다 조금씩 갱신된다.
- 목표 검색엔진은 한국 Google, 일본 Google, Yahoo! Japan이다.
- 공개 업데이트 기록 페이지는 만들지 않는다. 변경 내역은 로컬 `CHANGELOG.md`에만 기록한다.
- SEO용 긴 설명 글이 필요해지면 사용자에게 작성 또는 검토를 요청한다.

## 첫 SSG 페이지

일본어는 기본 주소를 사용한다. 한국어는 `/ko/` 아래에 둔다.

| 정보 | 일본어 | 한국어 |
| --- | --- | --- |
| 첫 화면 | `/rs_dev/` | `/rs_dev/ko/` |
| 일반 장비 개방 옵션 | `/rs_dev/open-options/` | `/rs_dev/ko/open-options/` |
| 비규격 장비 일반 옵션 | `/rs_dev/instandard/options/` | `/rs_dev/ko/instandard/options/` |
| 비규격 장비 개방 옵션 | `/rs_dev/instandard/open-options/` | `/rs_dev/ko/instandard/open-options/` |
| 티어별 옵션 수치 | `/rs_dev/instandard/tier-values/` | `/rs_dev/ko/instandard/tier-values/` |

일본어 `/ja/` 주소는 만들지 않는다. 일본어 기본 주소와 내용이 같은 페이지가 두 개 생기면
검색엔진이 중복 페이지로 볼 수 있기 때문이다.

## 구현 방식

- Astro가 정적 HTML 생성과 페이지별 메타데이터를 맡는다.
- 기존 React 컴포넌트는 검색, 필터, 모달, 테마처럼 클릭이 필요한 부분에 재사용한다.
- 빌드할 때 핵심 제목, 설명, 옵션 표, 데이터 기준일을 HTML에 넣는다.
- 큰 CSV/JSON 전체를 모든 페이지에 넣지 않는다. 필요한 화면에서만 불러온다.
- 언어 버튼은 같은 내용의 다른 언어 주소로 이동한다.

## 페이지 공통 요소

각 페이지에는 다음을 넣는다.

- 언어에 맞는 제목과 짧은 설명
- 실제 옵션 정보 표 또는 핵심 요약
- `일본 클라이언트 기준`과 `CHANGELOG.md`의 마지막 게임 데이터 갱신 날짜
- 기능 요약과 데이터에서 자동으로 고른 대표 옵션 3개
- 한국어·일본어 페이지를 연결하는 언어 버튼
- 관련 정보 페이지 링크
- 페이지별 `title`, 설명, 정식 주소(canonical), 공유용 Open Graph 정보

## 검색엔진 파일

빌드 결과에 다음 파일을 만든다.

- `sitemap.xml`: 검색엔진이 찾을 수 있는 페이지 목록
- `robots.txt`: 크롤링 안내
- 언어 연결 정보(`hreflang`): 한국어·일본어 번역 페이지 관계

구조화 데이터는 사이트 정보와 현재 위치 표시만 사용한다. 실제 질문과 답변이 없는
FAQ 구조화 데이터는 사용하지 않는다.

## 배포 뒤 작업

1. GitHub Pages에 새 정적 파일을 배포한다.
2. 브라우저의 페이지 소스에 옵션 제목과 설명이 실제로 있는지 확인한다.
3. Google Search Console에 `https://nwon2464.github.io/rs_dev/`를 URL 접두어 속성으로 등록한다.
4. `https://nwon2464.github.io/rs_dev/sitemap.xml`을 제출한다.
5. 한국어와 일본어 주요 페이지의 색인 요청을 한다.
6. 유입 검색어를 보고 중요한 옵션의 상세 페이지를 추가한다.

## 구현 순서

1. Astro 기반 SSG 구조를 추가한다.
2. 첫 핵심 페이지를 한국어·일본어로 만든다.
3. 기존 React 탐색 기능을 새 주소에 연결한다.
4. 사이트맵, robots, 언어별 메타데이터를 추가한다.
5. GitHub Pages용 빌드와 생성 결과를 검증한다.
