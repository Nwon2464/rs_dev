# 낫 비규격 옵션 전체 목록

## 조사 기준

- 대상 장비는 `item_group_id: 54`, `item_group_name: "낫"`입니다.
- 옵션 범위는 `equipment[item_group_id=54].option_ids`에 들어 있는 ID만 사용했습니다.
- `enabled: true`인 티어만 “등장 가능한 티어”로 보았습니다. `enabled: false`는 웹에서도 숨깁니다.
- 모든 낫 옵션의 한국어 템플릿은 `[0]` 하나만 사용합니다. 별도 값 바인딩도 없습니다.
- 원본 `rolls`의 한 칸은 `[value_0, value_1, value_2]` 형태입니다. 낫 데이터의 모든 칸은 `[표시 수치, 0, 0]`입니다.
- 따라서 아래 “원본 롤 값”은 각 칸의 `value_0`을 원본 순서대로 적은 것입니다. `value_1`과 `value_2`는 모든 칸에서 `0`입니다.
- “웹 표시값”은 웹 코드가 중복을 제거한 뒤 보여 주는 고유 수치입니다.
- “원본 빈도”는 같은 수치가 원본 10칸에 몇 번 들어 있는지를 센 것입니다.
- 이 빈도는 원본 구조를 보존하기 위한 정보입니다. 실제 확률 가중치를 뜻한다고 단정하지 않습니다.
- `[n]`은 웹의 옵션 제목에서 “숫자가 들어갈 자리”를 뜻합니다.

## 웹에서 데이터를 사용하는 흐름

1. `web/src/main.tsx`가 비규격 `catalog.json`과 한국어 `base_options.json`을 읽습니다.
2. `web/src/components/instandard/InstandardOptionViewer.tsx`가 낫의 `option_ids`로 옵션을 고릅니다.
3. `web/src/domain/openOptions/instandardRendering.ts`가 활성 티어의 `rolls`를 순회합니다.
4. `web/src/domain/openOptions/renderTemplate.ts`가 `[0]` 자리에 `value_0`을 넣습니다.
5. `tierCandidateValues()`의 `Set`이 같은 표시값을 한 번만 남깁니다.

- 웹 표시값: `[10, 12, 14, 16]`
- 원본 롤 값: `[10, 10, 10, 12, 12, 14, 14, 14, 14, 16]`
- 원본 빈도: `10×3, 12×2, 14×4, 16×1`
