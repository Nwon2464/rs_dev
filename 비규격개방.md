# 비규격 장비 4줄 개방란 매핑

## 결론

34개 비규격 장비군은 모두 기존 일반 장비 개방 버킷 10개에 유일하게 연결된다. processed 자료에서 비규격 전용 1~4줄 행은 발견되지 않았고, 장비군 키로 연결된 기존 버킷의 `option_id`·`value_raw`(수치)·`option_tier`(등급)·변환기 유형·확률 행이 비규격에도 적용되는 매핑으로 확인된다. 따라서 매핑 기준으로는 일반 장비와 동일하다. 다만 독립된 비규격 레코드와의 값 대 값 비교는 아니다.

## 1~4줄 필드 비교

`equipment_converter_type_options.csv`에서 사용되는 10개 버킷의 7,326행을 검사했다. 모든 버킷에 개방 1·2·3·4줄이 있고, `option_id`, `value_raw`, `option_tier`, `converter_type`, `converter_probability`가 모두 채워져 있으며 `probability_sum_valid=true`였다. 변환기는 기존 분류 규칙인 일반·개량된·모조·불타는 변환기를 그대로 사용한다.

## 근거와 미확정

- 비규격 장비군: `data/processed/instandard_equipment.json` 안의 34개 `item_group_id`.
- 기존 개방 버킷·줄·옵션·수치·등급·변환기·확률: `data/processed/equipment_converter_type_options.csv`.
- 직접 조인 키: 두 산출물에 공통으로 보존된 `simpleGameText.dat item_group_id`; 옵션은 기존 파서의 `capa.dat option_id` 조인을 재사용했다.
- 재현: `python3 'scripts/[test]_trace_instandard_open_slots.py'`.
- 미확정 장비군: 없음. 단, 비규격 전용 개방 테이블이 별도로 존재한다는 증거는 processed JSON·CSV에 없으며, 실제 게임 실행 코드가 동일 버킷을 선택하는 지점은 확인하지 못했다.

원본 DAT는 조사하지 않았다. 기존 processed 산출물과 파서의 버킷 정의로 34개 모두가 유일하게 해결되었기 때문이다.
